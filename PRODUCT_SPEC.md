# 漫游册 V0.1 产品规格

## 产品目标

在两周内上线一个可访问的 Web MVP，让用户可以真实填写旅行需求、获得高质量 AI 攻略、修改某一天并保存一次结果。

当前交付范围是少量真实用户可访问的公网 Beta 基础：真实 AI、持久化、只读分享、匿名所有权、限额和反馈。

## 核心用户流程

1. 用户从首页了解产品并点击“开始规划”。
2. 用户填写目的地、日期、天数、人数、预算。
3. 用户选择一个旅行风格和最多三个兴趣。
4. 系统校验输入并进入生成状态页。
5. 生成完成后进入持久化的 `/trip/[id]` 攻略页。
6. 创建者可以调整某一天并撤销最近一次修改，分享访客只能查看。

## 页面规划

| 路由 | 页面 | V0.1 今晚状态 |
|---|---|---|
| `/` | 产品首页、示例预览、开始规划 | 已实现 |
| `/create` | 旅行需求与偏好表单 | 已实现 |
| `/generating` | 真实AI生成步骤 | 已实现 |
| `/trip/[id]` | 持久化真实AI攻略 | 已实现 |
| `/api/trips/[id]/revise-day` | 只修改目标DayPlan | 已实现 |

## 表单范围

极简必填：单一主目的地、天数（1–7）、旅行风格。用户可直接生成。

旅行风格单选：特种兵、慢旅行、懒人旅行、美食探索、情侣浪漫、亲子旅行。

折叠选填：日期确定程度、预算模式和口径、最多3项优先需求、出发城市、成人/儿童、交通偏好、周边一日游偏好、300字补充要求。

目的地区域展示静态旅行灵感，不声称实时热门或实时季节推荐。

## 暂不开发

- 登录、会员、支付
- 微信小程序、iOS App
- PDF、社区、多人协作
- 地图、实时路线、实时天气
- 酒店预订、门票和复杂数据源
- 未知目的地推荐和完整多城市路线

## 已知限制与待确认项

- 数据库是最终数据源；`localStorage`只用于表单与网络失败缓存，V0.1旧输入会迁移。
- 未填写日期时不产生具体日期；未填写预算时不产生精确费用承诺。
- 当前生成采用一次同步服务端请求；Vercel Node.js Fluid Compute 当前上限覆盖现有耗时。
- 支持最近一次单日修改撤销并写回数据库，暂无完整版本历史。

## 公网 Beta 持久化（2026-07-13）

- 数据库成为最终数据源，匿名 HTTP-only Cookie 识别创建者；localStorage 降级为缓存。
- 结果页为 `/trip/[id]`，UUID 链接持有者只读；修改和撤销必须通过服务端所有权校验。
- `/api/recent-trip` 恢复同一浏览器的最近攻略；单日修改采用 version 乐观锁。
- 每日完整生成和单日修改次数由环境变量控制，同一访客不能并发完整生成。
- 可选 Beta 访问码和 AI 总开关均在服务端生效；结果页增加最小反馈模块。
