# 一键出发产品原则

1. 用户能少填，就不要让他多填。
2. AI 能决定，就不要让用户决定。
3. 每一个功能，都要让用户更快出发。
4. 不做攻略工具，只做出发助手。
5. 用户能少看，就不要让他多读。

第五条意味着：短标签优先于长段落，视觉层级优先于重复说明，默认只展示结论与最重要信息，次要内容折叠；不展示内部枚举、系统术语和重复提示，不为了显得专业增加阅读负担。

更底层的产品观是：凡是可以由系统完成的认知劳动和重复劳动，都不应继续留给用户。预算、同行人、作息时间、强烈偏好和关键安全限制仍由用户表达，并保留修改权；系统完成其余大部分判断。

## 设计与开发检查

- 新字段是否真的会显著改变结果？否则不要让用户填写。
- 新文案能否缩成一句？次要说明能否折叠？
- 新功能是否缩短“想到旅行”到“可以出发”的距离？
- 错误是否保留输入并给出下一步，而不是暴露内部机制？
- 用户可见内容是否包含枚举、技术名词或重复结论？
