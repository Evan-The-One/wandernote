# 精美旅行海报

## 产品边界

文字攻略继续负责规划、修改与执行；“保存今日行程为图片”继续负责单日离线查看；旅行海报只在结果确认后由创建者主动生成，用于整趟旅行的保存与分享。旧 `premium_trip` 模板入口已经下线，历史记录不删除。

## 技术方案

V2 采用“确定性时间轴 + 逐活动视觉”混合方案。服务端把当前 `TripPlan` 按每页最多2天、每天最多7个条目整理；`gpt-image-2` 每天生成一张无文字的活动素材板，服务端使用 Sharp 将其裁成独立的3:2小图。浏览器 Canvas 只负责米白画布、左右双栏、深绿色 Day 标题、垂直时间轴、中文标题、提示、页码和品牌，不生成整页城市背景。

页数规则固定为 `ceil(days / 2)`：1天1张，2天1张，3天2张，4天2张，5天3张，6天3张，7天4张。偶数页左右双栏，奇数最后一天全宽单栏，不生成纯封面。输出固定1024×1536 JPEG，支持逐张保存、全部保存和系统分享。

活动视觉缓存复用现有 `trip_image_tasks.output_json`。缓存键包含国家/地区、目的地、地点名、活动类别、视觉版本、模型和Prompt版本；新任务读取最近成功的V2任务并复用匹配素材，不新增图片表。任务分析事件记录生成素材数、复用素材数与估算成本。

## 权限、额度与幂等

- V2 使用独立 `travel_poster_v2` 权益，每个匿名访客赠送1次；旧模板和V1海报额度不会占用V2机会。
- 只有行程创建者可创建任务；分享访客只能读取成功结果。
- 缓存键包含 trip、version、imageType、aspectRatio 与 version；相同版本重复请求复用任务。
- 任务先写入 `running`，成功后保存结构化内容和压缩 WebP 底图；失败写入安全错误码并自动返还权益。
- 请求经过同源校验、访客 Cookie、IP 风险限制、全站 AI 预算熔断和数据库访客锁；API Key 只存在服务端。

## 成本

生产使用 `OPENAI_IMAGE_MODEL=gpt-image-2`。活动素材板固定使用 low 质量；每个有未缓存素材的旅行日调用一次，官方当前参考输出成本约0.005美元/张素材板。成本按实际调用数记录到任务和 `ai_usage` 分析事件。

## 环境变量

```env
OPENAI_IMAGE_MODEL=gpt-image-2
OPENAI_IMAGE_QUALITY=medium
FREE_LIFETIME_PREMIUM_IMAGE_LIMIT=1
```

## 回滚

应用可以回滚到本轮安全检查点。无需删除 `trip_image_tasks` 或 `entitlement_ledger`；`premium_trip` 与 `travel_poster` 通过 `image_type` 和独立权益类型隔离。不得为了回滚删除历史任务。
