# Development Log

## 2026-07-13 · “一键出发”品牌与核心体验改版

- 正式品牌统一为“一键出发”，口号为“不用查攻略，一键直接出发。”，覆盖首页、导航、Metadata、等待、结果、修改、分享、反馈和错误状态。
- 首页首屏直接承载目的地、天数和旅行风格三项输入；`/create` 继续兼容旧链接。
- 增加固定旅行灵感和静态目的地推荐面板，不接入实时数据。
- 等待页展示本次选择摘要与风格规划说明；结果页先解释偏好理解、节奏、活动密度、步行、交通、住宿和预算。
- 保持 `slow` 与 `lazy` 独立：前者强调停留更久与深入体验，后者进一步减少活动、步行和换乘，并增加晚出发与休息。
- 单日修改快捷指令、撤销、分享只读和反馈文案完成品牌化；逐日反馈复用现有反馈表，不修改历史数据。
- 前端外链审计未发现 Google Fonts、外部图片、第三方脚本、外部 CSS 或客户端 OpenAI 调用。
- 中国大陆访问限制主要来自 `.vercel.app` 与 Vercel 海外基础设施；自定义域名只能降低域名被限制概率，不能保证可用性。

## 2026-07-13 · 公网 Beta 基础

- 建立安全 Git 基线，空白化环境变量示例并增加密钥扫描。
- 选择 Neon PostgreSQL + Drizzle，新增五张数据表、JSONB、迁移与乐观锁。
- 生成、读取、单日修改、一次撤销、最近攻略、只读分享和反馈改为数据库流程。
- 增加匿名安全 Cookie、每日限额、并发锁、请求大小限制、AI 总开关和可选 Beta 访问码。
- 选择 Vercel 同步方案：Node.js Fluid Compute Hobby 300 秒上限覆盖当前任务；接口设置 120 秒。
- 完成 Neon 迁移和真实持久化测试，代码已推送至 `Evan-The-One/wandernote` 并由 Vercel 自动部署。
- 公网 Beta 地址为 `https://wandernote-beryl.vercel.app`；关闭 Vercel 登录保护后，未登录访客可以访问。
- 生产验收：真实 AI 生成 22 秒，单日修改 14 秒；刷新持久化、匿名创建者权限、分享只读、反馈保存和撤销恢复均通过。
- 部署时发现 `neon-http` 不支持交互式事务，改为普通查询使用 HTTP、原子写入使用按请求创建并关闭的 Neon WebSocket 事务连接。
- 初次 Vercel `DATABASE_URL` 导入错误；修正过程中连接串曾出现在工具输出，已立即轮换 Neon 密码并更新 Vercel，旧凭据已失效。

## 2026-07-12 · V0.1单日修改与等待体验

### 完成内容

- 新增`POST /api/trips/revise-day`，模型只接收目标日、全局约束和相邻日摘要，只返回更新后的DayPlan。
- 新增单日修改Prompt、Zod响应协议和独立质量校验，失败后定向修复一次。
- 攻略页增加移动端底部修改面板、快捷指令、原页轻量等待、变化摘要和一次撤销。
- 修改后的攻略与最近一次撤销快照保存在localStorage，刷新后仍可撤销。
- 生成页增加等待秒数、五段稳定文案、离开提示、失败重试与输入保留。
- 增加生成和修改分段耗时日志，不记录Key、目的地或补充要求全文。

### 性能分析

- 完整生成基线五组平均约48.2秒；解析、Zod和质量校验通常合计低于5毫秒，主要耗时来自模型生成和失败后的第二次修复调用。
- 尝试限制文案长度后五组平均53.4秒，没有收益，已撤回。
- 单日修改成功案例13–21秒，平均约16秒；局部上下文显著减少了输出和等待时间。

### 下一步计划

- 数据库阶段保存TripPlan及DayPlan修订，替代localStorage单设备方案。
- 增加服务端任务状态后，可把等待文案升级为真实阶段进度。

### 最终验收

- 五组真实单日修改测试通过：杭州、上海、厦门和苏州成功修改；成都高消费要求在第二轮测试中以原日预算内替换成功。
- 单日修改最终一轮耗时12–17秒，平均13.6秒，且均未触发修复调用。
- 移动端完成真实流程：打开第二天修改面板、提交自然语言、原页等待、展示变化摘要、确认其他日期不变、撤销、刷新后确认原行程持久化。
- 完整生成并发测速在本机代理下不稳定：苏州27秒成功，杭州和厦门超时。顺序生成五组此前全部成功，V0.1继续限制重复提交，不支持并发生成。
